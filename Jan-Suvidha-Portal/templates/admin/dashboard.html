SMS Reminder Data Flow

Step 1: User registers → stored in MongoDB users collection (phone, name, language, village)

Step 2: User answers questions → Rule Engine checks eligibility → stored in applications collection

Step 3: Admin views dashboard → sees Eligible vs Applied per village/scheme

Step 4: Admin clicks "Send Reminders" → queries applications where status = eligible_not_applied

Step 5: For each user: get phone from users → send SMS via Fast2SMS API → log in sms_logs

Step 6: Mark reminder_sent = true on that application record

No AI is used in this flow. It is 100% rule-based and deterministic.
